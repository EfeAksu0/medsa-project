1. Abstract

Medysa is a comprehensive, full-stack web application designed to address the primary cause of failure in financial markets: psychological error and lack of data-driven decision making. Unlike traditional trading journals that simply record data, Medysa integrates a "Human-in-the-Loop" AI coaching system that analyzes user behavior in real-time. By leveraging Large Language Models (LLMs) and statistical analysis, the platform detects "Tilt" (emotional instability), provides corrective coaching, and offers institutional-grade analytics. This paper details the system architecture, core algorithms, and implementation of the Medysa platform.

2. Problem Statement

Retail traders face a failure rate of approximately 90-95%. Research indicates this is rarely due to a lack of technical knowledge, but rather:

  1. Emotional volatility: "Revenge trading" and "FOMO" (Fear Of Missing Out) lead to impulsive decisions.
  2. Lack of Feedback Loops: Traders often repeat mistakes because they lack objective, data-driven feedback on their performance.
  3. Data Fragmentation: Trading data is often scattered across various brokerages with poor visualization tools.

Medysa solves these issues by acting as an external compliance officer and psychologist, automating the feedback loop that professional proprietary trading firms provide manually.

3. System Architecture & Technology Stack

The application is built using a modern decoupled architecture to ensure scalability, security, and real-time performance.

3.1 Frontend (Client-Side)

  • Framework: Next.js 14 (React) using the App Router for server-side rendering and optimized SEO.
  • Language: TypeScript for type safety and robust development.
  • Styling: TailwindCSS with a custom "Glassmorphism" design system, utilizing heavily customized UI components (Lucide icons, Framer Motion for animations).
  • State Management: React Context API and SWR for real-time data fetching.

3.2 Backend (Server-Side)

  • Runtime: Node.js with Express.js.
  • Language: TypeScript.
  • Authentication: Dual-layer authentication using Supabase Auth (JWT) integrated with custom middleware.
  • AI Integration: Google Gemini Pro via REST API, utilizing a prompt-engineering architecture to simulate different coaching personas.

3.3 Database & Storage

  • Database: PostgreSQL (hosted on Supabase).
  • ORM: Prisma, providing a type-safe interface for complex relational queries (Users -> Trades -> Journals).
  • Triggers: Custom PL/pgSQL triggers for automated user provisioning and data integrity.

4. Key Features & Implementation

4.1 The AI "Psychologist" Engine

The core innovation of Medysa is the AI Coach (AiCoach.tsx / aiService.ts). It does not simply chat; it performs a forensic analysis of the user's recent data.

  • Context Assembly: When a user queries the AI, the backend aggregates their last 15 trades, recent journal entries, and overall win/loss stats into a UserContext object.
  • Tilt Detection Algorithm: A heuristic algorithm scans for risk factors:
    • Loss Streaks: Detects consecutive losses > 3.
    • Drawdown Severity: Flags losses that exceed 2.5x the average loss.
    • Rapid Execution: Identifies "over-trading" patterns.
  • Dynamic Personas: Based on the calculated "Tilt Score," the AI dynamically switches personality:
    • The Medic: Engages when risk is CRITICAL. Tone is soothing and protective.
    • The Sniper: Engages when the user is WINNING. Tone is disciplined to prevent overconfidence.
    • The Forensic: Default analytical mode for data review.

4.2 Institutional Analytics Dashboard

The dashboard (dashboard/page.tsx) provides real-time visualization of trading performance.

  • Metrics: Calculates Profit Factor, Win Rate, and Average Risk/Reward ratio in real-time.
  • Visualizations: Interactive charts built with Recharts to show PnL curves and daily performance heatmaps.
  • Journaling: A rich-text entry system allows traders to log potential setups (Draft), active trades (Open), and completed reviews (Closed), creating a searchable database of their own psychology.

4.3 Content Creation Engine ("Reel Generator")

Recognizing the modern "Creator Economy," Medysa includes a unique marketing tool (reel/page.tsx).

  • Function: It programmatically generates short-form video content (TikTok/Reels) using CSS animations and DOM manipulation.
  • Implementation: It utilizes a sequenced state machine to animate text, charts, and AI reactions, allowing users (and the platform admin) to produce viral marketing content without video editing software.

5. Security & Deployment

  • Row-Level Security (RLS): While Prisma handles application logic, database-level security ensures users can only access their own financial data.
  • Environment Safety: Strict separation of production and development keys (Stripe, Gemini, Supabase).
  • CI/CD: Automated deployment pipelines via Vercel for the frontend.

6. Conclusion

Medysa represents a significant step forward in retail trading tools. By treating "Trading Psychology" as a data science problem rather than a vague concept, it provides tangible, actionable value to users. The project successfully demonstrates the integration of Generative AI into a high-stakes decision-making environment, moving beyond simple chatbots to create a context-aware performance coach.
